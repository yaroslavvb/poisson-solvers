The problem

Discretize −u″(x) = f(x) on (0,1) with u(0) = u(1) = 0 on n = 256 interior points: A = tridiag(−1, 2, −1)/h² with h = 1/257. In the original source configuration b is zero except b₁ = +1 and b₂₅₆ = −1 (a + and a − point charge next to the grounded ends; the midpoint entry is set to 0 exactly as in the original code, a deliberate no-op), and the exact potential (dashed gray in the first panel) is piecewise linear. The eigenvalues of A are λk = (2 − 2cos(kπh))/h², so the condition number is κ = cot²(πh/2) ≈ 2.7×10⁴.

The four panels

Identical to the Mathematica Manipulate dashboard, with axes frozen over the whole history so nothing rescales while you scrub: current solution u (blue, with the exact solution dashed), current residual r = b − Au (red), the step Δu taken at this iteration (green), and the convergence tracker (log residual, red marker at the current step). The tracker overlays all five solvers — the selected one is highlighted.

The solvers, and how they relate

  • Richardson: x ← x + ωr with a fixed step ω. This is gradient descent with a fixed learning rate on the energy ½xᵀAx − bᵀx. Shown with the conservative ω = 1/λmax; the eigencomponent of the residual at λ contracts by |1 − ωλ| per step — up to 1 − 1/κ for the lowest mode, hence worst-case O(κ) iterations.
  • Jacobi: x ← x + D⁻¹r. Here the diagonal is constant, D = (2/h²)I, so Jacobi is exactly Richardson with the optimal fixed step ω* = 2/(λmin+λmax) = h²/2 — mode k contracts by |cos kπh| per step: fast mid-spectrum, excruciating at both ends. Still O(κ).
  • Gradient descent (steepest descent): the same direction r, but with the exact line search α = rᵀr∕rᵀAr recomputed each step. Adaptive, yet the worst-case rate (κ−1)/(κ+1) is the same O(κ) class — watch it zig-zag. (Fun fact: with the point-charge source the ±charge symmetry makes b’s eigen-spectrum symmetric about the middle eigenvalue 2/h², so the exact line search evaluates to exactly h²/2 every step — steepest descent literally reproduces Jacobi, iteration for iteration.)
  • SOR: a Gauss–Seidel sweep with over-relaxation. At the optimal ω* = 2/(1+sin πh) ≈ 1.976 the spectral radius drops to ω*−1, turning O(κ) into O(√κ). Try the ω slider: at ω = 1 (plain Gauss–Seidel) the low modes crawl, and the optimum is razor-sharp.
  • Conjugate gradient: also O(√κ), but parameter-free and optimal — it minimizes the A-norm error over the entire Krylov space at every step, and in exact arithmetic terminates in at most n steps. Watch the solution panel with the point-charge source: information propagates inward from the two sources roughly one grid cell per iteration.

Two lessons hiding in the numbers

The right-hand side matters as much as the matrix. With the original point charges, every method stops within a few hundred iterations, far below the worst-case O(κ) counts. No accident: sources at nodes adjacent to a Dirichlet boundary have eigen-expansion coefficients ∝ sin(kπh) — almost no energy in the slow extreme modes, lots in the mid-spectrum modes that every method damps quickly. Switch the source to the two charged slabs, whose energy sits in the slow low-frequency modes, and the textbook hierarchy appears in full: Richardson and Jacobi crawl through tens of thousands of iterations, gradient descent zig-zags in the same O(κ) class, while SOR(ω*) and CG finish orders of magnitude sooner — that is why the tracker’s x-axis defaults to log scale.

A small residual is not a small error. The last table column compares each stopped solver against the exact solution u* = A⁻¹b. At the same stopping criterion ‖r‖/‖b‖ ≤ 10⁻², the methods land at wildly different solution errors (with point charges: a few percent for SOR(ω*) versus ~80% for Jacobi/Richardson). The error is e = A⁻¹r, so residual components in low modes get amplified by up to 1/λmin — with κ ≈ 2.7×10⁴, a “converged” residual can hide a mostly unconverged potential.
